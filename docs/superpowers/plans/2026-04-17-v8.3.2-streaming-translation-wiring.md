Player uses MediaSource Extensions (MSE) to chain per-segment opus audio seamlessly. GPU fleet is woken by `POST /api/translate/warmup` via `streaming_sessions.gpu_warm` flag; daemon unions batch + streaming + warmup when deciding to spin up Vast.ai/RunPod; watchdog reclaims stuck streaming segments. Bilingual navigable side panel lets Qing (and Bosco) jump to cue pairs in either language.

**Tech Stack:** Python 3.14 (API + worker), Flask + flask-sock (WebSocket), SQLite WAL, faster-whisper (STT on Vast.ai SSH tunnel or RunPod HTTPS), DeepL Pro (translation), edge-tts `zh-CN-XiaoxiaoNeural` (CPU TTS), systemd, ffmpeg (segment split + chapter concat), MediaSource Extensions + Opus (browser audio), vanilla JS, 18 locale JSON catalogs.

## Phase B — Three-Tier Priority Queue

### Task 6: Docs update — priority model

- [ ] **Step 1: Edit `docs/STREAMING-TRANSLATION.md`** — priority section:

```
Priority levels (lower = higher urgency):
  0  P0 — cursor buffer fill. Populates first ~3 minutes (6 segments)
         forward of the cursor. Must flow before playback resumes.
  1  P1 — forward chase. Continues producing segments past the cursor
         buffer toward end-of-chapter / next logical break. Deprioritized
         only if the user jumps/stops.
  2  P2 — back-fill. Produces segments between prior translated tail and
         the cursor. Runs after P0 is satisfied so the side panel and
         future backward-scrubbing have continuous context.

On seek-beyond-buffer: existing pending segments downgraded to P2; six
segments forward of the new cursor promoted/inserted at P0; end-of-chapter
remainder queued at P1; gap between prior tail and new cursor queued at P2.

On stop: all pending segments downgraded to P2 (back-fill preserves work
for future resume / side-panel completeness).
```

```python
BUFFER_AHEAD_SEGMENTS = 6  # ≈3 min at 30 s/segment

def handle_seek_impl(conn, audiobook_id, locale, chapter_index, segment_index):
    """Three-tier priority rebalance on seek.

    Existing semantics deprecated: 2-tier (demote all, promote 6 forward).
    New semantics:
      - Tier 0 (P0): segments [t .. t+BUFFER_AHEAD-1] — cursor buffer.
      - Tier 1 (P1): segments > t+BUFFER_AHEAD-1 — forward chase.
      - Tier 2 (P2): segments < t — back-fill.
    Already-processing segments are left alone (worker already claimed them).
    """
    t = segment_index
    hi = t + BUFFER_AHEAD_SEGMENTS - 1
    # Clear priority on pending rows for this (book, chapter, locale) first
    conn.execute(
        "UPDATE streaming_segments SET priority = 2 "
        "WHERE audiobook_id=? AND chapter_index=? AND locale=? "
        "AND state='pending'",
        (audiobook_id, chapter_index, locale),
    )
    # Promote cursor buffer to P0
    conn.execute(
        "UPDATE streaming_segments SET priority = 0 "
        "WHERE audiobook_id=? AND chapter_index=? AND locale=? "
        "AND state='pending' AND segment_index BETWEEN ? AND ?",
        (audiobook_id, chapter_index, locale, t, hi),
    )
    # Promote forward chase to P1
    conn.execute(
        "UPDATE streaming_segments SET priority = 1 "
        "WHERE audiobook_id=? AND chapter_index=? AND locale=? "
        "AND state='pending' AND segment_index > ?",
        (audiobook_id, chapter_index, locale, hi),
    )
    conn.commit()
```

```javascript
class MseAudioChain {
  constructor(audioEl) {
    this.audio = audioEl;
    this.mediaSource = new MediaSource();
    this.audio.src = URL.createObjectURL(this.mediaSource);
    this.queue = [];
    this.ready = new Promise(res => {
      this.mediaSource.addEventListener('sourceopen', () => {
        this.sourceBuffer = this.mediaSource.addSourceBuffer(
          'audio/ogg; codecs=opus'
        );
        this.sourceBuffer.mode = 'sequence';
        this.sourceBuffer.addEventListener('updateend', () => this._drain());
        res();
      }, { once: true });
    });
  }
  async enqueueSegment(url) {
    await this.ready;
    const buf = await (await fetch(url)).arrayBuffer();
    this.queue.push(buf);
    this._drain();
  }
  _drain() {
    if (!this.sourceBuffer || this.sourceBuffer.updating) return;
    const next = this.queue.shift();
    if (next) this.sourceBuffer.appendBuffer(next);
  }
  endOfStream() {
    if (this.mediaSource.readyState === 'open') this.mediaSource.endOfStream();
  }
}
```

```python
def _derive_phase(conn, audiobook_id, locale):
    row = conn.execute(
        "SELECT "
        " SUM(state='pending') AS pending, "
        " SUM(state='processing') AS processing, "
        " SUM(state='completed') AS completed, "
        " SUM(state='failed') AS failed "
        "FROM streaming_segments "
        "WHERE audiobook_id=? AND locale=?",
        (audiobook_id, locale),
    ).fetchone()
    session = conn.execute(
        "SELECT state, gpu_warm FROM streaming_sessions "
        "WHERE audiobook_id=? AND locale=? "
        "ORDER BY requested_at DESC LIMIT 1",
        (audiobook_id, locale),
    ).fetchone()
    if row["failed"] and row["failed"] > 0:
        return "error"
    if row["completed"] and row["completed"] >= BUFFER_AHEAD_SEGMENTS:
        return "streaming"
    if row["processing"] and row["processing"] > 0:
        return "buffering"
    if session and session["gpu_warm"] and session["state"] == "warmup":
        return "gpu_provisioning"
    if (row["pending"] or 0) > 0:
        return "warmup"
    return "idle"
```

```javascript
async function loadChapterPairs(bookId, ch, locale) {
  const [source, target] = await Promise.all([
    fetch(`/api/subtitles/${bookId}/${ch}/source`).then(r => r.text()),
    fetch(`/api/subtitles/${bookId}/${ch}/${locale}`).then(r => r.text()),
  ]);
  const srcCues = parseVtt(source);
  const tgtCues = parseVtt(target);
  return pairVttCues(srcCues, tgtCues);
}

function pairVttCues(src, tgt) {
  // Pair by time-window overlap (not strict 1:1 — translation can merge/split cues).
  const out = [];
  let j = 0;
  for (const s of src) {
    const pair = [s, []];
    while (j < tgt.length && tgt[j].start < s.end) {
      pair[1].push(tgt[j]);
      j += 1;
    }
    out.push(pair);
  }
  return out;
}

function render(pairs) {
  const srcOl = panel.querySelector('.source-cues');
  const tgtOl = panel.querySelector('.target-cues');
  for (const [s, ts] of pairs) {
    const li1 = document.createElement('li');
    li1.textContent = s.text;
    li1.dataset.start = s.start;
    li1.addEventListener('click', () => audio.currentTime = s.start);
    srcOl.append(li1);

    li2.textContent = ts.map(t => t.text).join(' ');
    li2.dataset.start = s.start;
    li2.addEventListener('click', () => audio.currentTime = s.start);
    tgtOl.append(li2);
  }
}
```

## Glossary

- **P0/P1/P2**: streaming_segments priority tiers (0 highest urgency).
- **Buffer threshold**: 6 completed segments (≈3 min) before playback transitions from `buffering` to `streaming`.
- **Segment**: 30-second slice of a chapter at a fixed offset; unit of worker processing.
- **Chapter consolidation**: after all chapter segments complete, their VTTs merge into `chapter_subtitles` and their opus files concat into `chapter_translations_audio` (shared cache used by BOTH batch and streaming pipelines).
- **Wake**: `POST /api/translate/warmup` inserts a `streaming_sessions` row with `gpu_warm=1, state='warmup'`; daemon counts this in `get_total_pending` for 15 min.
- **MSE**: `MediaSource` + `SourceBuffer` — browser API for feeding per-segment audio into a single `<audio>` element seamlessly.
- **Bilingual panel**: two-column cue list (source + target) with click-to-seek and current-cue highlight.
