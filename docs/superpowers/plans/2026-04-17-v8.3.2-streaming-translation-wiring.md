# v8.3.2 Streaming Translation Wiring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish wiring the streaming translation pipeline that v8.3.0/v8.3.1 shipped as an orphaned front-half — add the missing systemd unit, install/upgrade/manifest plumbing, and phase-aware UX so users see concrete progress instead of a static "排队中" spinner.

**Architecture:** The streaming worker `scripts/stream-translate-worker.py` already exists but has zero wiring into systemd, install.sh, upgrade.sh, or the install manifest. Segments inserted by `POST /api/translate/stream` into the `streaming_segments` table are never consumed. Fix: add a dedicated `audiobook-stream-translate.service` unit that runs the worker continuously as `audiobooks:audiobooks`, register it in all three plumbing locations, and add phase reporting to the API + UI so the buffering overlay shows "Processing segment N of M" / "Waking GPU…" / specific error states instead of a generic spinner.

**Tech Stack:** Python 3.14 (worker, API), Flask (coordinator API), systemd, SQLite (streaming_segments / streaming_sessions), vanilla JS (buffering overlay), i18n JSON catalogs (18 locales).

---

## Scope Boundary (READ BEFORE STARTING)

This release is correctly small because 8.3.0/8.3.1 already shipped the majority of the streaming pipeline. This plan covers only the pieces they forgot:

- **IN scope**: systemd unit for the worker; install/upgrade/manifest entries; phase reporting on the API; phase-aware messages in the overlay; locale strings; tests; QA cycle; staged release.
- **OUT of scope**: GPU wake-on-demand provisioning (that's a separate architectural item in `project_streaming_pipeline_design.md`); refactoring the streaming protocol; changing the segment size or buffer threshold; adding new translation providers.

If something feels out of scope while executing, stop and confirm before widening.

Phase taxonomy (surfaced to the UI buffering overlay):
    queued     — segment rows exist but none are processing/completed/failed
    processing — at least one segment has state='processing'
    buffering  — completed < threshold AND no processing row
    ready      — completed >= threshold
    error      — at least one segment has state='failed'
               (error wins over other states so the UI can surface it)
"""

import pytest

from library.backend.api_modular.streaming_translate import (
    BUFFER_THRESHOLD,
    _derive_phase,
)


def _rows(**counts):
    """Build a list of fake row dicts for the given state counts."""
    rows = []
    for state, n in counts.items():
        for i in range(n):
            rows.append({"state": state, "segment_index": i})
    return rows


def test_no_rows_returns_queued():
    assert _derive_phase([], BUFFER_THRESHOLD)["phase"] == "queued"


def test_all_pending_returns_queued():

    assert phase["current_segment"] is None

def _derive_phase(rows, buffer_threshold: int) -> dict:
    """Derive the user-facing phase from a list of streaming_segments rows.

    Args:
        rows: iterable of dict-like objects with 'state' and 'segment_index'.
        buffer_threshold: segments needed before playback can start.

    Returns:
        {
            'phase': one of 'queued' | 'processing' | 'buffering' | 'ready' | 'error',
            'current_segment': segment_index of a currently-processing row, or None.
        }
    """
    completed = 0
    processing_idx = None
    has_failed = False
    has_any = False
    for r in rows:
        has_any = True
        s = r["state"]
        if s == "failed":
            has_failed = True
        elif s == "processing" and processing_idx is None:
            processing_idx = r["segment_index"]
        elif s == "completed":
            completed += 1

    if has_failed:
        return {"phase": "error", "current_segment": processing_idx}
    if processing_idx is not None:
        return {"phase": "processing", "current_segment": processing_idx}
    if not has_any:
        return {"phase": "queued", "current_segment": None}
    if completed >= buffer_threshold:
        return {"phase": "ready", "current_segment": None}
    if completed > 0:
        return {"phase": "buffering", "current_segment": None}
    return {"phase": "queued", "current_segment": None}
```

  // Map server phase strings to localized message keys.
  // UI fallback text is used only if i18n hasn't loaded yet.
  function phaseMessage(phase, currentSegment) {
    var key, fallback;
    switch (phase) {
      case "processing":
        if (typeof currentSegment === "number") {
          key = "streaming.phase.processingSegment";
          fallback = "Processing segment " + (currentSegment + 1) + "...";
          return typeof t === "function"
            ? t(key, { current: currentSegment + 1 })
            : fallback;
        }
        key = "streaming.phase.processing";
        fallback = "Processing audio...";
        break;
      case "buffering":
        key = "streaming.phase.buffering";
        fallback = "Buffering translated audio...";
        break;
      case "ready":
        key = "streaming.phase.ready";
        fallback = "Ready — starting playback...";
        break;
      case "error":
        key = "streaming.phase.error";
        fallback = "Translation failed. Retrying...";
        break;
      case "queued":
      default:
        key = "streaming.phase.queued";
        fallback = "Preparing translation...";
        break;
    }
    return typeof t === "function" ? t(key) : fallback;
  }
```
